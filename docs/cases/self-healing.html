고장나면 스스로 고칩니다

API가 바뀌어 스킬이 멈춰도, AI가 원인을 진단하고 코드를 수정합니다.

자동 복구
시나리오

어느 날 갑자기 API가 바뀌면

1

잘 돌아가던 스킬이 실패합니다

외부 API의 응답 포맷이 바뀌거나, 엔드포인트가 변경되면 스킬 실행이 실패합니다.

❌

스킬 실행 실패

macro-economy-report — TypeError: Cannot read property 'regularMarketPrice' of undefined

오전 8:01
2

AI가 에러를 분석합니다

엔진이 실패한 코드와 에러 메시지를 LLM에게 전달합니다. LLM이 원인을 진단하고 수정 코드를 생성합니다.

자동 진단 — AI
// 원인: Yahoo Finance API 응답 구조 변경 // regularMarketPrice → currentPrice 으로 필드명 변경
- const price = data.regularMarketPrice; + const price = data.currentPrice ?? data.regularMarketPrice;
3

자동으로 수정하고 재실행합니다

Full autonomy 모드에서는 수정을 즉시 적용하고, Supervised 모드에서는 승인을 요청합니다.

🔧

자동 수정 적용됨

macro-economy-report: 필드명 변경 (regularMarketPrice → currentPrice)

오전 8:02
✅

재실행 성공

macro-economy-report — 정상 동작. 텔레그램 전송 완료.

오전 8:02
4

아무 일 없었다는 듯, 브리핑이 도착합니다

사용자는 실패와 복구 과정을 알 필요 없습니다. 결과만 도착하면 됩니다.
